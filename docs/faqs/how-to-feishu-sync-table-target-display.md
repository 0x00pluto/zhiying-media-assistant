# 飞书同步表格链接识别与目标确认

## 分享链接 vs 地址栏链接

飞书多维表格有两种常见链接形式：

| 来源 | 典型格式 | 是否含 `table=` |
|------|----------|-----------------|
| 点击「分享」复制的链接 | `/wiki/{token}?from=from_copylink` | 通常**没有** |
| 浏览器地址栏完整链接 | `/wiki/{token}?table=tbl...&view=vew...` | **有** |

同一 Wiki 节点 token 可以对应多张数据表。分享链接只指向多维表格文档，**不会指定具体数据表**。当文档内有多张表时，扩展无法自动判断同步目标，需要在弹窗中手动选择，或改用地址栏完整链接。

### 推荐做法

1. **首选**：在飞书中打开目标数据表，从浏览器地址栏复制含 `?table=tbl...` 的完整链接。
2. **备选**：粘贴分享链接后，在同步弹窗下方的「请选择要同步的数据表」下拉框中选择目标表。

### 示例

- 分享链接（需选表）：`https://xxx.feishu.cn/wiki/WusEwh...?from=from_copylink`
- 地址栏链接（可直接识别）：`https://xxx.feishu.cn/wiki/WusEwh...?table=tbld18JU20YknsfE&view=vewf76rIdQ`

## 四入口独立缓存

笔记详情、批量笔记、批量博主、批量评论各自使用独立 `storageKey`，互不影响。同步弹窗输入框下方会显示「文档名 · 数据表名」供二次确认。

## 常见错误

| 副标题提示 | 原因 | 处理 |
|------------|------|------|
| 分享链接未指定数据表，请在下方选择… | 多表 + 分享链接无 `table=` | 在下拉框选表，或换地址栏链接 |
| 当前 Wiki 链接不是多维表格… | 链接指向知识库目录/文档 | 打开多维表格页面后复制地址栏链接 |
| 请先配置飞书 App ID 与 App Secret | 选项页未配置 | 前往扩展设置页填写 |
| 链接中的数据表不存在… | `table=` 参数错误 | 从地址栏重新复制 |

## 飞书应用权限

若地址栏完整链接也无法同步，请检查：

1. 飞书开放平台应用已开通「查看、评论、编辑和管理多维表格」「查看知识空间节点信息」等权限。
2. 目标多维表格已通过「添加文档应用」将扩展应用加为**可管理**协作者。
3. 改动 background 或权限后，在 `chrome://extensions` **重新加载**扩展。
